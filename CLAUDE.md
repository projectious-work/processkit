# CLAUDE.md — processkit

**processkit** is the content layer for [aibox](https://github.com/projectious-work/aibox).
It ships the process primitives (18 building blocks), skills (multi-artifact
packages), process templates, and MCP servers that aibox installs into projects.

See [README.md](README.md) for the overview and the bootstrap-loop explanation.

## Critical: src/ vs everything else

**Everything under `src/` is shipped content.** Treat it like source code. It
is what `aibox init` installs into consuming projects when pinned to a
processkit tag.

**Everything outside `src/` is about THIS repo:** its own dev environment, its
own project management, its own docs site, its own contributor tooling.

| Directory            | Role                                                     |
|----------------------|----------------------------------------------------------|
| `src/primitives/`    | **SHIPPED** — schemas, state machines, format spec       |
| `src/skills/`        | **SHIPPED** — multi-artifact skill packages              |
| `src/processes/`     | **SHIPPED** — process definitions                        |
| `src/packages/`      | **SHIPPED** — package tier definitions                   |
| `docs-site/`         | **SHIPPED** (via gh-pages) — user-facing documentation   |
| `context/`           | THIS repo's project-management artifacts                 |
| `context/processes/` | THIS repo's own process definitions (used by its agents) |
| `.claude/skills/`    | Skills the agent uses ON this repo (not shipped)         |
| `.devcontainer/`     | Generated by aibox 0.14.1 (not shipped)                  |

Do not confuse `src/skills/` (the content we ship) with `.claude/skills/` (the
agent tooling aibox gave us for working on this repo).

## Project structure

```
src/                    ← CONTENT SHIPPED TO CONSUMERS
  primitives/
    FORMAT.md           ← entity file format (apiVersion/kind/metadata/spec)
    schemas/            ← YAML schemas per primitive type
    state-machines/     ← default state machine definitions
  skills/
    <skill-name>/
      SKILL.md          ← three-level instructions (Level 1/2/3)
      examples/         ← example outputs
      templates/        ← entity scaffolding templates
      mcp/              ← Python MCP server source (optional)
  processes/            ← process definitions
  packages/             ← package tier definitions

docs-site/              ← Docusaurus site — user-facing docs (shipped via gh-pages)
context/                ← THIS project's own management artifacts
.claude/skills/         ← skills the agent uses ON this repo (not shipped)
.devcontainer/          ← generated by aibox 0.14.1 (not shipped)
```

## Working in this repo

**Building processkit content:** edit `skills/`, `primitives/`, `processes/`, `packages/`.
These are validated by `aibox lint` (once that lands) and released as git tags.

**Releasing:** semver tags. Phase 1 target: `v0.1.0` (foundation — schemas and format spec).

**aibox version pinning:** this repo pins `aibox 0.14.1` in `aibox.toml`. That version
still ships embedded skills, so it can bootstrap processkit without a chicken-and-egg
problem. Do not bump aibox here until processkit consumption logic lands in aibox AND
processkit has enough content to be consumable — then coordinate the upgrade deliberately.

## Context files

- `context/AIBOX.md` — aibox configuration notes (auto-generated from aibox.toml)
- `context/OWNER.md` — profile of the project owner
- `context/PRD.md` — product requirements
- `context/PROJECTS.md` — project registry
- `context/processes/` — process definitions for THIS repo (bug-fix, code-review, release)
