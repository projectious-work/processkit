# CLAUDE.md — processkit

**processkit** is the content layer for [aibox](https://github.com/projectious-work/aibox).
It ships **19 process primitives**, **106 skills** (multi-artifact packages),
process templates, six Python MCP servers, a shared Python lib, the
PROVENANCE.toml provenance map, and supporting scripts. aibox installs the
content into projects when consumers pin a processkit tag in their `aibox.toml`.

**Current tag:** v0.4.0 (2026-04-07).

## Read these first

If you are new to this repo, in this exact order:

1. **`README.md`** — what processkit is, how it relates to aibox, the bootstrap loop
2. **`context/HANDOVER.md`** — comprehensive briefing for incoming agents (17 sections covering everything from architectural decisions to gotchas)
3. **This file (CLAUDE.md)** — for the rule below + entry-point commands
4. **`context/BACKLOG.md`** — what's deferred, what's next (BACK-001..BACK-014 plus v0.4.0 done section)

After those four, dig into specific files as the work demands.

## Critical: src/ vs everything else

**Everything under `src/` is shipped content.** Treat it like source code. It
is what `aibox init` installs into consuming projects when pinned to a
processkit tag.

**Everything outside `src/` is about THIS repo:** its own dev environment, its
own project management, its own docs site, its own contributor tooling.

| Directory                  | Role                                                       |
|----------------------------|------------------------------------------------------------|
| `src/PROVENANCE.toml`      | **SHIPPED** — file → last-changed tag mapping (v0.4.0+)    |
| `src/primitives/`          | **SHIPPED** — schemas, state machines, format spec (19 kinds) |
| `src/skills/`              | **SHIPPED** — 106 multi-artifact skill packages            |
| `src/processes/`           | **SHIPPED** — process definitions (INDEX.md only as of v0.4.0; content in Phase 4) |
| `src/packages/`            | **SHIPPED** — 5 package tier definitions                   |
| `src/lib/processkit/`      | **SHIPPED** — internal Python lib used by MCP servers (NOT a primitive) |
| `docs-site/`               | **SHIPPED** (via gh-pages) — Docusaurus documentation     |
| `scripts/`                 | Developer tooling: `stamp-provenance.sh`, `processkit-diff.sh`, `smoke-test-servers.py` |
| `context/`                 | THIS repo's project-management artifacts                   |
| `context/processes/`       | THIS repo's own process definitions (used by its own agents) |
| `.claude/skills/`          | Skills the agent uses ON this repo (NOT shipped)           |
| `.devcontainer/`           | Generated by aibox — NOT shipped                           |

Do not confuse `src/skills/` (the content we ship) with `.claude/skills/` (the
agent tooling aibox gave us for working on this repo).

## The 6 MCP servers (v0.3.0+)

Six servers ship under `src/skills/<name>/mcp/server.py` — all uv-runnable
PEP 723 inline-deps Python scripts using the official MCP SDK over STDIO:

| Skill | Layer | Tools |
|---|---|---|
| `index-management` | 0 | `reindex`, `query_entities`, `get_entity`, `search_entities`, `query_events`, `list_errors`, `stats` |
| `id-management` | 0 | `generate_id`, `validate_id`, `list_used_ids`, `format_info` |
| `event-log` | 0 | `log_event`, `query_events`, `recent_events` |
| `workitem-management` | 2 | `create_workitem`, `transition_workitem`, `query_workitems`, `get_workitem`, `link_workitems` |
| `decision-record` | 2 | `record_decision`, `transition_decision`, `query_decisions`, `get_decision`, `supersede_decision`, `link_decision_to_workitem` |
| `binding-management` | 2 | `create_binding`, `end_binding`, `query_bindings`, `resolve_bindings_for` |

## Quick commands

```bash
# Smoke-test all 6 MCP servers (fastest feedback loop)
uv run scripts/smoke-test-servers.py

# Diff between two tags using PROVENANCE.toml at each tag
scripts/processkit-diff.sh --from v0.3.0 --to v0.4.0
scripts/processkit-diff.sh --from v0.3.0 --to v0.4.0 --format toml

# Regenerate PROVENANCE.toml against current git history (release time)
scripts/stamp-provenance.sh v0.X.Y

# Validate the PROVENANCE.toml is up to date (CI check, no write)
scripts/stamp-provenance.sh --check v0.X.Y
```

## Working in this repo

**Building processkit content:** edit `src/skills/`, `src/primitives/`,
`src/processes/`, `src/packages/`. These will be validated by `aibox lint`
once that lands (Phase 4.4 on the aibox side, in flight).

**Releasing:** semver tags. The full release process is documented in
`context/HANDOVER.md` section 14. Always run `scripts/stamp-provenance.sh`
before tagging — the diff script's correctness depends on it.

**aibox version pinning:** this repo pins `aibox 0.14.1` in `aibox.toml`.
That version still ships embedded skills, so it can bootstrap processkit
without a chicken-and-egg problem. **Do not bump aibox here until** (a) aibox
ships processkit-consumption logic AND (b) processkit has a tag that the new
aibox knows about. Even bumping to 0.14.2/0.14.3/0.14.4 (pure bug-fix
releases) is safe but is currently a deferred decision — see HANDOVER §2.

## Out of scope (read HANDOVER §15 for the full list)

Do NOT add: RBAC enforcement, signed commits, workflow execution engine,
deterministic event logging, container infrastructure, a `processkit` CLI,
or REST/gRPC tool interfaces. processkit is consumed (via git tags), it
does not run as a command. If you find yourself wanting any of these,
you are likely in the wrong project.
