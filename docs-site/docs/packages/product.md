# product

**Intended for:** end-to-end product development teams.
**Extends:** `software`

The most comprehensive tier. Software plus design, mobile, framework,
and product-specific skills. Use when engineering, design, research, and
operations all live in the same repository.

## What product adds on top of software

- **Design:** `frontend-design`, `mobile-app-design`, `logo-design`
- **Framework:** `fastapi-patterns`, `tailwind`, `typescript-patterns`,
  `flutter-development`, `reflex-python`
- **Data subset:** `data-visualization`, `feature-engineering`,
  `ai-fundamentals`, `prompt-engineering`, `llm-evaluation`
- **Language conventions:** `python-best-practices`, `rust-conventions`,
  `go-conventions`, `java-patterns`
- **Documentation:** `latex-authoring`, `excalidraw`, `infographics`
- **Product ops:** `seo-optimization`, `agent-management`

## When NOT to use product

If you don't actually need design, mobile, or framework skills — stick
with `software`. The product tier is large and expecting every team to
manage it is false economy.
